# short-fans Business SSOT - クリエーター流入戦略

## 位置づけ

- 初期 creator を `どこから / どうやって / 何を約束して` 取るかを整理する
- 市場論より一段実務に近い、`立ち上げ方` と `供給獲得` の前提を置く

## 安定した事実

- 初期 creator target は `英語圏の OnlyFans系 short 強者` とする
- 初期は `成人向けど真ん中` で行くが、`short 面` 自体は露骨なポルノにしない
- `X` は adult content policy が比較的明示されており、英語圏 adult creator の公開面として自然
- `Instagram` と `TikTok` は露骨な成人向け送客の主面には向かず、`選別面` として見る方が自然
- `Reddit` は NSFW community / profile conversion 面としては使えるが、`短尺中心` の洗練された feed の主面ではない
- 初期の creator 獲得は `営業` ではなく、`認知拡大からの自然流入` を前提にする
- `Reels / TikTok` の流行り音源をそのまま `short-fans` に持ち込める前提は置かない

## 継続仮説

- 主な creator 供給源は、`X` で短い見せ動画を継続運用している `OnlyFans / Fansly / Fanvue` creator
- `Instagram / TikTok` は接触面より、`short を作る力` と `見せ方の安定感` の確認面として使う方が精度が高い
- 初期 creator 流入は、`AIシーズニング` と `外部 short マーケ` で知名度を上げ、creator に `この面なら伸びる / 稼げる` と感じさせる方が筋がよい
- つまり creator 獲得の主戦略は `営業` ではなく `面の強さによる自発流入` である
- 獲得メッセージは `取り分` 単体ではなく、`同じ撮影 / 同じ main をもとに外部SNS用の流入用shortと short-fans 用 short を作り分けながら main unlock を増やせる` に置く方が強い
- 初期 feed の質を守るには、誰でも自由に押し込める状態より `承認制の creator 申請` に寄せた方がよい
- 初期 creator 集団は、`流行り音源依存` が低く、`音源権利に依存しない` 複数 short を作れる creator を優先した方がよい

## 初期 creator の選別基準

- 既存 monetization 面を持つ
- `X` で短い見せ動画を継続投稿している
- `Instagram` または `TikTok` で short distribution 実績がある
- `ポルノを見せない short` でも魅力を作れている
- vertical な `main` を継続制作できる
- `1 canonical main : 複数 short` の運用に向く
- 英語圏 audience を持つ
- 自前音声 / 話し声 / 環境音でも short の引きを成立させられる

## 初期流入源の優先順位

1. `AIシーズニング` と外部 short による認知拡大からの自然流入
2. `X` 上で `short-fans` を見た creator の自発申請
3. `Instagram / TikTok` で platform の見せ方を見た creator の自発申請
4. 初期 creator 集団からの紹介

## creator 向け約束価値

- `同じ撮影 / 同じ main をもとに、外部SNS用の流入用shortと short-fans 用 short を作り分けながら main unlock を増やせる`
- `short は sample ではなく、そのまま main の冒頭になる`
- `main` が縦型-only なので continuity が崩れない
- `再生数` ではなく `unlock` ベースで見返せる
- `成人向けど真ん中` だが `short 面` はかっこよく保てる
- `short-fans` 用の安全な音源レーンで short を作り分けられる

## 未解決論点

- `US中心` を正式前提にするか、`広い英語圏` を最初から含むか
- creator が自然流入するまでの初期面の厚みをどう作るか
- early creator に与えるインセンティブを何にするか
- `候補リスト` をどの指標で作るか
- launch 初期の target creator 数をいくつに置くか
- `流行り音源依存` を候補選定時にどう判別するか
