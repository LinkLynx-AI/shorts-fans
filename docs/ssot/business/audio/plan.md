# short-fans Business SSOT - Audio Plan

## 位置づけ

- `音源` 論点に対して、`MVP` から将来拡張までの段階的な計画を置く
- ここでは `いま何をやるか` と `後で何を取りにいくか` を分ける

## 現時点の計画

### Phase 1. まずは `外部面は音源あり / short-fans は audio-safe` で始める

- 初期は、creator に `音源ありの流入用short` を `Instagram / TikTok` など外部面で使ってもらう
- `short-fans` には、同じ撮影 / 同じ main に紐づく `audio-safe` な別shortを upload してもらう
- ここでいう `audio-safe` は次
  - 無音
  - original audio
  - 話し声 / 環境音
  - royalty-free
  - `short-fans` 側で用意した house audio

- つまり、初期案は `完全に音源なし only` ではなく、`major platform の流行り曲に依存しない音` へ寄せる、である

### Phase 1 で取るべき補助策

- `safe audio playbook`
  - どの音なら `short-fans` で通しやすいかを creator に渡す
- `edit support`
  - 流入用shortから `short-fans` 用 short を切り直す支援
- `house audio pack`
  - 低コストで共通の vibe を作るための自前パック

## 他の解決策

### 1. `音源なし` だけに固定しない

- `音源なし` だけだと creator の表現幅を削りすぎる
- 初期は `silent only` より、`original / spoken / commissioned / royalty-free` を入れた方がよい

### 2. `full license` の前に `limited licensed lane` を挟む

- いきなり `Reels` と同等の catalog を取りにいくのではなく
  - 1 territory
  - 1 use case
  - 限定 catalog
  - capped creator cohort

- で pilot する方が現実的

### 3. 先に `house audio pack` を作る

- `commissioned / acquired` の小さな pack を先に持てば
  - creator が安全に使える
  - `short-fans` 固有の vibe を作れる
  - rights-heavy deal に行く前に需要検証できる

- ので、最初の大きな候補である

## 将来計画

### Phase 2. traction が見えたら limited catalog を検討する

- まず取りにいくのは `Reels級の全曲開放` ではなく、`rights-cleared limited catalog` である
- 例えば
  - 特定 territory
  - 特定ジャンル
  - 上位 creator 限定
  - 尺制限つき

- のように scope を絞る

### Phase 3. 十分に伸びたら broader music licensing を取りにいく

- `short-fans` の user growth と monetization が十分に立った後に、major label / publisher を含む broader deal を検討する
- ただし、この段階でも `full Reels parity` を最初の目標にはしない
- 狙う順番は
  1. `house audio / commissioned pack`
  2. `limited rights-cleared catalog`
  3. `broader label / publisher deals`

## コスト観の整理

### いまやるコスト

- `safe audio playbook`
- creator edit support
- `house audio pack`

- は、`full catalog deal` より圧倒的に軽い
- ここは `ops` と `small catalog budget` で始められる余地がある

### 後で取りにいくコスト

- `Reels / TikTok` 級の recognizable catalog は、公開情報ベースでも `major platform economics` の話になる
- したがって、これを取りにいく判断は `機能追加` ではなく `大型資本配分` として扱うべき

## 現時点の推奨

- まずはユーザー案をベースに進めてよい
- ただし wording は
  - `こっちでは音源なしを上げてもらう`

- ではなく
  - `こっちでは audio-safe な別shortを上げてもらう`

- にした方が正確で、戦略の自由度も高い
- 将来の license も、`いきなり major full catalog` ではなく `limited licensed lane` を挟む前提で置く
- 例；複数short/1 mainのケースで、Reels / TikTokにはReels / TikTok公式的に使える音源を使用したShortを投稿、short-fansにはaudio-safeのshortを撮り投稿。
　- クリエーターの工数一つ増えるのでなるべく早くReels / TikTokと同じ音源をshort-fansに載せれるように契約を組みたい。

## 未解決論点

- `house audio pack / safe audio playbook / edit service` の優先順位
- limited catalog をやるなら、どの国とどの creator cohort から始めるか
- broader licensing に進む判断指標を `user growth / GMV / creator demand` のどこに置くか
