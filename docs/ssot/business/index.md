- 初期は `成人向けど真ん中` で行くが、`short 面` 自体は露骨なポルノにしない
- `rails` は `検証速度` より `確実性` を優先する
- creator 流入は `営業` ではなく、`AIシーズニング + short マーケ` による認知拡大からの自然流入を前提にする
- `business SSOT` は `strategy/`、`revenue/`、`audio/`、`data/` に分けて管理する
- `business SSOT` は `revenue-model`、`market-and-strategy`、`creator-acquisition-strategy`、`marketing`、`audio/`、`ai-seeding-strategy`、`data-strategy`、`data-products-and-governance` を主軸に整理する

## 継続仮説

- 収益の中心は `subscription` 単体ではなく、`PPV / tips / upsell` を含む複合課金になる可能性が高い
- `short -> 有料転換 -> 継続課金` の導線設計が、収益密度に強く効く
- 市場性の見極めでは、国別差分とカテゴリ差分が重要
- `public short feed` 自体は海外では既に競争領域であり、差別化は `short -> canonical main -> unlock` を product / analytics / creator workflow まで一貫させられるかに寄る
- `英語圏最大市場` を取りにいきつつ、`short -> canonical main -> unlock` のスタイルは崩さない
- 初期 creator 獲得の主戦略は、creator 直営業ではなく `面の強さと認知` による自発流入である

## 未解決論点

- `英語圏` の中で `US中心` と `広い英語圏` のどちらで始めるか
- `TAM / SAM / SOM` を `英語圏 adult short creator` のどこで切るか
- 国別の決済制約、法規制差分をどう織り込むか
- 競合比較で何を差別化軸として固定するか
- スケール戦略を `creator supply` と `fan demand` のどちらから作るか
- `adult-capable payment / MOR / KYC / age assurance` stack をどう組むか
- `流行り音源依存` の高い creator supply をどう扱うか
- `AIシーズニング` と `platform short マーケ` の役割分担をどう切るか
