# short-fans Product SSOT - Visual Direction

## 位置づけ

- `fan` 向け UI のリファレンス、動き、情報配置の原則をまとめる
- 色や演出の嗜好ではなく、`どの UI をベースにするか` の安定した判断を置く

## 安定した事実

- fan 向けの `feed / short detail / paywall` は `Instagram Reels` をベースに 9 割採用で設計する
- 動きと情報配置は `Instagram Reels` を基本にし、差分は `like / comment` を持たないことと `Unlock` / 購入処理まわりに絞る
- トーンは `明るめ` を基本に置く
- 視覚方向は `水色基調` の `モダン UI / UX` を基本に置く
- `paywall` も別世界の面にせず、`Reels` の文脈を崩さないことを優先する

## Instagram Reels との差分

- `like` は持たない
- `comment` は持たない
- `Unlock ¥X` を primary CTA として置く
- `creator block` は `Instagram Reels` と同じ考え方にし、`profile 入口` を別導線として分離しない
- `Unlock ¥X` は画面下部に置き、`creator block` より上に置く
- `main` の長さは CTA の外に出さず、`[ Unlock ¥X | n分 ]` の 1 つの pill 内に含める
- 説明文は `Instagram Reels` と同じように通常の caption を書ける形にする
- `creator profile` は `Instagram profile` に近い見せ方にし、short 一覧では `Unlock` を直接出さない

## 継続仮説

- `short -> main` の転換は、情報量を増やすより `既知の UI 文脈を保つこと` の方が効きやすい
- 初期は `Instagram Reels` に強く寄せた方が、学習コストと設計コストの両方を下げやすい
- `fan` 向け UI を先に固める方が、`creator` 側 UI を詰める前提も揃えやすい

## 未解決論点

- `PWA` 前提でどこまで `Reels` 級の体感に寄せるか
- `short` の末尾でどこまで suggestive な表現を許すか
- `離散系` や編集強めの動画をどこまで許容するか
