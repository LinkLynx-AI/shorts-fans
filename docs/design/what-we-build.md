# 何を作るか

## この文書の目的

この文書は、このリポジトリで作るサービスの輪郭を明文化するための設計書です。
まずは「何を作るか」を固定し、詳細な画面設計、契約、実装ルールはその後に分けて整理します。

## サービス概要

このサービスは、`shorts + アダルト` を前提にした短尺動画サービスです。

- 収益モデルやクリエイターとファンの関係性は、既存サービスでいう OnlyFans に近いです。
- ただし、UI / UX の骨格とコンテンツの入口は TikTok / YouTube Shorts のような短尺動画フィードを中心に設計します。
- OnlyFans を参考にするのは、月額購読、限定動画、課金導線など、アダルト機能を成立させる追加部分です。
- モバイルでは、YouTube のような下部バーで `home`、`shorts`、`subscriptions`、`profile` などの主要画面を切り替えられる構成を想定します。
- サイトに入ったときの初期表示は `shorts` にします。
- `shorts` は TikTok を強く参考にした縦型の連続視聴体験にします。
- `home` は MyFans のように、好みのクリエイター、新着動画、ジャンル別導線、検索などを扱う画面にします。
- `home` は、`上部バー（通知、検索）`、`おすすめジャンル`、`ピックアップ動画`、`ピックアップクリエイター`、`新着動画` の順に構成します。
- `home` は myfans と同様に縦スクロールを前提にします。
- `subscriptions` は、購読中クリエイターの新着や継続視聴を扱う画面にします。
- ユーザーはまず `shorts` で短尺動画を連続的に視聴し、気になったクリエイターが見つかったらその人のページへ遷移します。
- クリエイターページでは、その人の他の動画を見たり、課金によって限定動画へアクセスしたりできる設計を想定します。

## アプリ構成方針

- アプリ下部には、YouTube のようなナビゲーションバーを置きます。
- 下部バーには、`home`、`shorts`、`subscriptions`、`profile` を置きます。
- サイト訪問時の初期表示は `shorts` です。
- `shorts` は、TikTok を強く参考にした縦型の動画視聴画面です。
- `home` は、ユーザーが能動的にクリエイターや動画を探すための発見画面です。
- `home` には、好みのクリエイター、新着動画、ジャンル別導線、検索機能を持たせます。
- `home` の基本順序は、`上部バー（通知、検索）`、`おすすめジャンル`、`ピックアップ動画`、`ピックアップクリエイター`、`新着動画` です。
- `home` はスマホ前提の縦スクロール画面とし、PC でも縦長の構成を維持します。
- `subscriptions` は、購読中クリエイターの新着や継続視聴に素早くアクセスするための画面です。
- `profile` は、自分自身の情報や設定、投稿管理などを扱う画面です。

## 収益モデル方針

- 課金モデルは、クリエイターごとの月額購読とします。
- ユーザーは、特定のクリエイターを月額で購読することで、そのクリエイターの限定動画へアクセスします。
- 単品購入は、この時点では前提にしません。
- 収益源は月額購読だけでなく、広告も含めます。
- 広告は YouTube Shorts レベルで、視聴体験を不快にしない範囲に抑える方針です。

## 公開動画と限定動画の境界

- 公開動画は shorts 動画です。
- 公開動画は、少なくとも非購読ユーザーが見られるコンテンツとして扱います。
- 限定動画は、対象クリエイターを月額購読したユーザーだけが視聴できます。
- 非購読ユーザーには、限定動画そのものは見せず、ぼかされたサムネイルだけを表示します。
- 非購読ユーザーがそのぼかしサムネイルを押した場合は、月額購読を促す文章や導線を表示します。
- 体験イメージは TikTok のメンバーシップ導線に近いものを想定します。
- ただし、成人向けサービスとしての年齢確認や法的なアクセス制御は、この境界定義とは別に設計します。

## 動画フォーマット方針

- shorts 動画は短めの動画とし、長さの上限は YouTube Shorts の上限時間に合わせます。
- 限定動画は時間無制限とします。
- ただし、shorts 動画と限定動画はどちらも縦型動画に統一します。
- 理由は、主な視聴環境をスマホと想定しており、そのほうが UX が良いと判断しているためです。

## 体験の中心

このサービスの中心体験は、次の流れです。

1. ユーザーが短尺動画フィードを開く。
2. フィード上で複数のクリエイターの動画に触れる。
3. 気になるクリエイターを見つけたらクリエイターページへ移動する。
4. クリエイターページで他の公開動画と、ぼかしサムネ付きの限定動画導線を見る。
5. ぼかしサムネを押して、月額購読を促す導線を見る。
6. そのクリエイターを月額購読する。
7. 購読後、縦型の限定動画を視聴する。

## このサービスの立ち位置

### OnlyFans に近い点

- クリエイター単位でファンがつく
- クリエイターごとに有料コンテンツへ誘導する
- 限定コンテンツの閲覧に課金が関わる

### TikTok / YouTube Shorts に近い点

- サービスの入口が短尺動画フィードである
- まずコンテンツに触れ、その後にクリエイターへ興味を持つ流れである
- UI の主戦場が一覧ページではなく、縦型の連続視聴体験である
- `shorts` 画面は TikTok を強く参考にした UI / UX にする

### OnlyFans に近いのはどこか

- クリエイターごとの月額購読で限定コンテンツにアクセスする点
- 限定動画を paywall の内側に置く点
- 非購読ユーザーに対して課金導線を明確に見せる点

## 現時点で確定していること

- サービスのジャンルは「短尺動画 x アダルト」です。
- アプリ下部には、`home`、`shorts`、`subscriptions`、`profile` を切り替えるナビゲーションバーがあります。
- サイトに入ったときの初期表示は `shorts` です。
- `shorts` は TikTok を強く参考にした縦型連続視聴体験です。
- `home` は、好みのクリエイター、新着動画、ジャンル別導線、検索を扱う発見画面です。
- `home` の基本順序は、`上部バー（通知、検索）`、`おすすめジャンル`、`ピックアップ動画`、`ピックアップクリエイター`、`新着動画` です。
- `home` は myfans と同様に縦スクロールを前提とし、PC でも縦長構成を維持します。
- `subscriptions` は、購読中クリエイターの新着や継続視聴を扱う画面です。
- 入口となる主要体験は、`shorts` で短尺動画を連続視聴することです。

- shorts 動画の長さ上限は YouTube Shorts の上限時間を基準にします。

- 広告は、YouTube Shorts レベルで不快にならない視聴体験を目指します。
- ユーザーは気になったクリエイターに対して、公開動画から有料コンテンツへ遷移できる設計にします。

## この文書ではまだ固定しないこと

この段階では、以下は未確定として扱います。

- フィードの推薦ロジック
- `home` の各セクションで何件見せるか
- `home` の各カードに何の情報を載せるか
- `home` の各セクションを押したときの遷移先
- `subscriptions` に並べる情報の単位
  - クリエイター一覧を主にするか、新着動画フィードを主にするか
- クリエイターページの詳細構成
- 課金を促す文章、モーダル、画面遷移の具体的な UI
- 広告の表示位置、頻度、フォーマット
- クリエイター登録、投稿、運営管理の詳細フロー
- コメント、いいね、DM などのコミュニケーション機能の有無
- 決済、本人確認、年齢確認、モデレーションの具体的な運用設計

## 今後この文書から分離する設計テーマ

- 画面一覧と主要導線
- ユーザー種別ごとのユースケース
- コンテンツの公開範囲と権限モデル
- 課金導線と限定コンテンツの扱い
- クリエイター体験とファン体験の責務分離
