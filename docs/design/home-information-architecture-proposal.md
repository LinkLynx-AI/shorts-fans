# home 画面の情報設計提案

## この文書の位置づけ

この文書は確定仕様ではなく、myfans の公開情報を調査した上で作る提案書です。
`home` をどういう順序と見せ方で構成するかを決めるためのたたき台として扱います。

## 調査で確認できたこと

### 1. myfans には home から使う検索導線がある

公式ガイドでは、`ホームの右上にある虫眼鏡マーク` から検索を始めると説明されています。
検索対象は `ジャンル`、`タグ`、`クリエイター`、`投稿` です。

ここから言えるのは、myfans の home は単なるおすすめ一覧ではなく、探索の入口として `検索` が強い位置にあるということです。

### 2. myfans にはジャンル別ランキングがある

公式ガイドでは、検索結果から `〇〇のランキングを見る` を選んでジャンル別ランキングへ移動できると説明されています。
また別の記事では、`ジャンルごとのクリエイターおよび投稿ランキング` を楽しめるとされています。

ここから言えるのは、myfans の発見体験は `検索` と `ランキング` が強く結びついていることです。

### 3. myfans には home から見に行く保存済み・視聴履歴がある

公式ガイドでは、`ホーム` から `保存済み` を開けること、`ホーム` から `視聴履歴` を開けることが説明されています。

ここから言えるのは、myfans の home は `新しいものを探す場所` だけでなく、`過去に見たものへ戻る場所` としても機能しているということです。

### 4. myfans には表示内容を切り替える指向設定がある

公式ガイドでは、`ホーム` 右上の性別マークから `一般アダルト / ゲイ・BL / 全年齢` を切り替えられると説明されています。
英語版ガイドでは、同種の設定が `Explore` ページの右上ギアから行えるとされています。

ここは表記ゆれがありますが、少なくとも myfans には `探索画面の上部で表示対象を切り替える仕組み` があると見てよいです。

### 5. myfans には無料投稿の新着面がある

公式ガイドでは、`新着投稿` に表示される条件が説明されており、無料プラン投稿または完全無料投稿が対象とされています。

ここから言えるのは、myfans は `新着` を独立した発見軸として持っており、かつ新着面には無料で触れられる投稿を流している可能性が高いということです。

## 調査から導ける設計方針

このサービスの `home` は、myfans に近づけるなら次の役割を持つべきです。

- `検索の入口`
- `ジャンル / ランキングの入口`
- `新着無料投稿の入口`
- `過去の保存済み / 視聴履歴への入口`
- `表示対象の切り替え入口`

つまり、`home` は単なるおすすめ面ではなく、`探索ハブ` と `回遊ハブ` を兼ねる画面にするのが筋です。

## 現時点の合意事項

このスレッド時点では、次を合意事項として扱います。

1. `home` の順序は `上部バー（通知、検索）`、`おすすめジャンル`、`ピックアップ動画`、`ピックアップクリエイター`、`新着動画`
2. `home` は myfans と同様に縦スクロール
3. スマホでの利用を前提とし、PC でも縦長構成を維持する

## これまでの調査からの提案との関係

もともとの提案では、検索、指向切り替え、ランキング、保存済み / 視聴履歴までを home の主導線に強く含める案を出していました。
ただし、現時点の合意では、まず `発見性の高い主セクション` を優先して 1 画面目を軽くする方向に寄せています。

この整理は妥当です。
理由は、home のワイヤー段階では `何を主役にするか` を先に切るべきであり、補助導線まで 1 画面目に詰め込むと役割がぼけるからです。

## セクションごとの提案

### 1. 上部バー

最上部には次を置きます。

- サービスロゴ
- 右上通知アイコン
- 右上検索アイコン

この段階では、上部バーは `通知` と `検索` を優先します。
myfans の公開情報で確認できた `検索導線の強さ` を引き継ぎつつ、通知はサービス継続率に効く導線として上部固定に置くのが自然です。

### 2. おすすめジャンル

ここでは、ジャンルを発見の最初の入口として見せます。

- 横スクロールの genre chips
- 人気ジャンルを先頭に置く
- ジャンル押下で、そのジャンルの動画やクリエイター一覧へ遷移させる

myfans の公開情報でも、ジャンルは検索とランキングの軸です。
したがって、home のかなり上部に置くのは筋が通っています。

### 3. ピックアップ動画

ここは home の主役セクションです。

- 無料で触れられる動画を中心に並べる
- 並び順はおすすめベースでもよいが、少なくとも編集的な `pickup` に見える構成にする
- 各カードにクリエイター名とジャンルを出す

新着を先に出すより `pickup` を先に出すのは妥当です。
理由は、home は shorts と違って受動視聴の場ではないので、まず `今見てほしい面` を出したほうが画面の意図が明快になるからです。

### 4. ピックアップクリエイター

ここは `好みのクリエイター` を扱うセクションとして解釈できます。

- 横スクロールのクリエイターカード
- アイコン、名前、短い紹介、ジャンル
- `おすすめ` や `人気` の理由を短く見せる

ここを動画より下に置くのは自然です。
先にコンテンツを見せてから、作り手を掘らせる順のほうが視覚的な引きが強いからです。

### 5. 新着動画

新着は `pickup` の後ろに置くのがよいです。

- 無料で触れられる shorts を中心に並べる
- まずは新着順
- 各カードにクリエイター名とジャンルを出す

myfans の `新着投稿` が無料投稿中心であることと整合します。

## 今回の合意では 1 画面目から外れたもの

次の要素は価値がありますが、今回の 1 画面ワイヤーの主構成からは外れました。

- 指向切り替え
- ランキング
- 保存済み
- 視聴履歴

これは悪い判断ではありません。
主役導線を先に確定し、補助導線は後から足すほうがワイヤーが崩れにくいからです。

## 推奨レイアウト

モバイルでは次の順を推します。

1. 上部バー
2. おすすめジャンル
3. ピックアップ動画
4. ピックアップクリエイター
5. 新着動画

## この順序にする理由

- 最初に `通知 / 検索` という全体導線を置く
- 次に `ジャンル` で探索の入口を置く
- その後に `動画` を主役として見せる
- 次に `クリエイター` を掘らせる
- 最後に `新着` で更新性を見せる

この順序にすると、`home` は `shorts` の補助ではなく、独立した探索画面として成立します。

## この段階でまだ決めないこと

- `おすすめジャンル` を何件見せるか
- `ピックアップ動画` と `新着動画` を何件見せるか
- `ピックアップクリエイター` の抽出基準
- 通知を何の通知まで含めるか
- 指向切り替えを home 1 画面目のどこへ戻すか
- ランキングや視聴履歴を home に戻すか、別導線へ寄せるか

## 私の提案

現時点の合意を踏まえると、`home` は次の一文で定義できます。

`通知と検索を上部に持ち、ジャンル、ピックアップ動画、ピックアップクリエイター、新着動画を縦に積んだ探索ハブ`

この定義にしておくと、`shorts` は受動的な視聴、`home` は能動的な探索、`subscriptions` は継続視聴、`profile` は自分の管理、という役割分担がきれいに切れます。
